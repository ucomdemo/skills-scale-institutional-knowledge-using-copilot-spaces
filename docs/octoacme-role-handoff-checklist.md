# OctoAcme — Role Handoff Checklist

## Purpose
Ensure smooth transitions of ownership or information between roles during key project stages. Use this checklist whenever work moves from one role or team to another (e.g., BA to Developers, Developers to QA, QA to Release Manager).

---

## BA → Developers Handoff

- [ ] Acceptance criteria written and reviewed with PdM
- [ ] User stories groomed and estimated in the backlog
- [ ] Open questions or ambiguities documented and resolved
- [ ] Relevant wireframes or design specs linked (if applicable)
- [ ] Stakeholder sign-off on functional requirements obtained

---

## UX Designer → Developers Handoff

- [ ] Final design specs and assets shared (e.g., Figma link or exported assets)
- [ ] Interaction states and edge cases documented
- [ ] Accessibility requirements noted (colour contrast, keyboard navigation, ARIA labels)
- [ ] Open design questions flagged with proposed resolution
- [ ] Developer walkthrough of designs completed

---

## Developers → QA Handoff

- [ ] Feature branch merged to integration/staging environment
- [ ] Automated tests (unit + integration) passing in CI
- [ ] Known limitations or deferred items documented
- [ ] Acceptance criteria confirmed by developer before handoff
- [ ] Test data or environment setup notes provided to QA

---

## QA → Release Manager Handoff

- [ ] QA sign-off documented (test results or pass/fail summary)
- [ ] Release-blocking defects resolved or waived with justification
- [ ] Regression suite run and results shared
- [ ] UAT completed (if applicable) and sign-off recorded
- [ ] Outstanding non-blocking issues logged in the backlog for follow-up

---

## Release Manager → Stakeholders / Sponsors Post-Release Handoff

- [ ] Release notes published and shared
- [ ] Post-deploy verification results documented
- [ ] Any incidents or rollbacks noted with status
- [ ] Next steps or follow-on work communicated
- [ ] Post-release retrospective scheduled (if applicable)
